# mergeについて


mergeを実行する際のオプションについての整理。
下記3点のオプションがあるが、基本的には「--no-ff」を使うことになるかと。
理由としては、「ファーストフォワード」をしないことで「トピックブランチ」での変更履歴を残すためである。
下記のURLを参考にしました。
http://d.hatena.ne.jp/sinsoku/20111025/1319497900

①--ff
デフォルトの設定。
マージ対象とffの関係にある際は、マージコミットを作らずにブランチの参照先更新のみを行う。
ログが一本になり、「トピックブランチがあったという情報がなくなる」。
「トピックブランチが存在した」という情報を残す必要がない際はこのオプションを使うとよい。

git merge topic

②--no-ff
ファーストフォワードの関係でも、必ずマージコミットを作るオプション
--ffと違い「トピックブランチが存在した」という情報が残る。
基本的には「--no--ff」を利用したほうが良い

git merge --no-ff topic

③--squash
ワークツリーとインデックスについてはマージ後の状態になるものの、マージコミットを作らないためリモートリポジトリへの影響はない。
マージコミットが作られないため、後ほど自分でコミットしないといけない。
「トピックブランチが存在した」という情報も「コミットログ」も残らない。

git merge --squash topic
